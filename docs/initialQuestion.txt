De kern: een goede examen‑voorbereidingssite voor geschiedenis combineert twee sporen — feiten automatiseren én verbanden begrijpen. Dat vraagt om didactische principes die beide niveaus tegelijk aanspreken, en een web‑architectuur die dat ondersteunt op GitHub Pages.

🎯 Kernprincipe
De meest effectieve aanpak is een drie‑lagen didactisch model:

Micro‑kennis — namen, jaartallen, begrippen automatiseren

Meso‑verbanden — thema’s, causaliteit, chronologie, geografische spreiding

Macro‑structuur — grote lijnen, tijdvakken, ontwikkelingen, continuïteit/verandering

Je website moet elk van deze lagen ondersteunen met een eigen werkvorm.

📚 1. Didactische principes voor feitenkennis (micro)
RemNote - Notes & Flashcards (Android) — Price History & Similar | AppAgg
Elementary Black History Month Montessori Social Emotional Flash Card ...
Spaced repetition — herhaling op slimme intervallen verhoogt retentie enorm.

Retrieval practice — actief ophalen van kennis werkt beter dan lezen.

Immediate feedback — direct zien of iets goed is, versterkt leren.

Interleaving — door elkaar oefenen van thema’s voorkomt oppervlakkig leren.

Micro‑quizzes — korte, snelle vragen die je in 10–20 seconden beantwoordt.

Hoe implementeer je dit op een statische GitHub‑site?
Een flashcard‑module met JSON‑data (vraag, antwoord, hint).

Een adaptive quiz die moeilijke vragen vaker stelt (jij vroeg dit eerder al — perfect toepasbaar).

Een progress tracker in localStorage (geen backend nodig).

Een “daily 10” functie: elke dag 10 willekeurige feiten.

🌍 2. Didactische principes voor verbanden (meso)
47,049 Diagram Timeline Major Milestones Microscopy Stock Vectors and ...
Adam's Synchronological Chart Or Map Of History - Historical Timeline ...
Do you need your students to understand how WWI impacted the map of ...
Texas - State, Population & Map | HISTORY
Causale ketens — oorzaak → gevolg → lange‑termijn effect.

Thema‑gericht leren — politiek, economie, cultuur, religie, wetenschap.

Chronologische structuren — tijdlijnen, periodes, turning points.

Geografische context — kaarten, routes, verspreiding van ideeën.

Vergelijkend denken — twee gebeurtenissen naast elkaar leggen.

Hoe implementeer je dit?
Interactieve tijdlijn (bijv. JSON → vis.js of een eigen SVG).

Leaflet‑kaart met lagen voor perioden, thema’s, personen.

Causale diagrammen die je kunt klikken (oorzaak → gevolg).

Thema‑filters: politiek, economie, religie, cultuur, wetenschap.

“Explain mode”: klik op een gebeurtenis → krijg context, oorzaken, gevolgen.

🧠 3. Didactische principes voor grote structuren (macro)
Continuïteit en verandering

Turning points

Longue durée

Meta‑concepten — macht, identiteit, conflict, migratie, technologie.

Periodisering — waarom delen we geschiedenis zo in?

Implementatie‑ideeën
Een “big picture” dashboard met:

tijdvakken

kernconcepten

grote ontwikkelingen

koppelingen naar detailpagina’s

Een mindmap‑module (JSON → dynamische boomstructuur).

Een “compare periods” tool: twee tijdvakken naast elkaar.

🎯 Kernidee van de adaptieve quiz
De quiz gebruikt drie principes:

Retrieval practice

Spaced repetition

Adaptive weighting

Elke vraag krijgt een gewicht:

basisgewicht = 1

+1 voor elke fout

−1 voor elke 3 goede antwoorden (minimaal 1)

Zo ontstaat een dynamische lijst waaruit de quiz willekeurig maar gewogen kiest.

🧠 adaptive_quiz.js (volledige module)
Dit is een complete, werkende adaptieve quizmodule:

javascript


Copy
let QUIZ_DATA = [];
let currentQuestion = null;

// Stats opslaan in localStorage
function loadStats() {
  return JSON.parse(localStorage.getItem("quizStats") || "{}");
}

function saveStats(stats) {
  localStorage.setItem("quizStats", JSON.stringify(stats));
}

// Gewicht berekenen
function getWeight(q, stats) {
  const s = stats[q.question] || { correct: 0, wrong: 0 };
  let weight = 1 + s.wrong - Math.floor(s.correct / 3);
  return Math.max(1, weight);
}

// Gewogen lijst genereren
function buildWeightedList() {
  const stats = loadStats();
  const weighted = [];

  QUIZ_DATA.forEach(q => {
    const w = getWeight(q, stats);
    for (let i = 0; i < w; i++) {
      weighted.push(q);
    }
  });

  return weighted;
}

// Nieuwe vraag kiezen
function pickQuestion() {
  const weighted = buildWeightedList();
  return weighted[Math.floor(Math.random() * weighted.length)];
}

// UI elementen
const questionEl = document.getElementById("question");
const answerInput = document.getElementById("answerInput");
const feedbackEl = document.getElementById("feedback");

// Quiz starten
document.getElementById("startQuiz").onclick = () => {
  document.getElementById("quizContainer").style.display = "block";
  startQuiz();
};

function startQuiz() {
  fetch("../data/facts.json")
    .then(r => r.json())
    .then(json => {
      QUIZ_DATA = json;
      nextQuestion();
    });
}

function nextQuestion() {
  currentQuestion = pickQuestion();
  questionEl.textContent = currentQuestion.question;
  answerInput.value = "";
  feedbackEl.textContent = "";
}

// Antwoord controleren
document.getElementById("submitAnswer").onclick = () => {
  const userAnswer = answerInput.value.trim().toLowerCase();
  const correct = currentQuestion.answer.toLowerCase();

  const stats = loadStats();
  if (!stats[currentQuestion.question]) {
    stats[currentQuestion.question] = { correct: 0, wrong: 0 };
  }

  if (userAnswer === correct) {
    stats[currentQuestion.question].correct++;
    feedbackEl.textContent = "Goed!";
    feedbackEl.style.color = "green";

    saveStats(stats);

    setTimeout(nextQuestion, 700);
  } else {
    stats[currentQuestion.question].wrong++;
    feedbackEl.textContent = `Fout — het juiste antwoord is: ${currentQuestion.answer}`;
    feedbackEl.style.color = "red";

    saveStats(stats);

    setTimeout(nextQuestion, 1500);
  }
};
🧪 Hoe werkt dit didactisch?
Moeilijke vragen (veel fouten) → krijgen meer gewicht → komen vaker terug

Makkelijke vragen (veel goed) → krijgen minder gewicht → komen minder terug

Je leert dus sneller waar je zwak bent

Je bouwt duurzame feitenkennis op

Je hoeft geen backend → alles werkt in GitHub Pages
